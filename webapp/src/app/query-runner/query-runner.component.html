SPINDLE allows to train a logistic regression model across the datasets. Then, the trained model is used to predict the outcome of some row.
Each dataset is kept private by a dedicated node, and never shared with the other participants of the network. The trained model itself is not revealed to anyone but can be evaluated with a row that only the querier knows.
In the next tab, you can choose the parameters of the model to be trained, and the row to predict. The result will be shown in the last tab.

    • Learning rate: how much each iteration is to be taken into account. The greater, the faster it will obtain a solution but the more likely the solution won't be the best one.
    • Elastic rate: how much randomness to add at each local iteration, to try to find a better solution that the previous one.
    • Local iteration count: how many times the local model is trained before doing a network iteration. The greater, the slower it will compute locally but, if the datasets are similar, it would require less network iteration.
    • Local batch size: when computing a local iteration, how many rows to train upon at the same time. The greater, the more precise the solution will be but the more memory it will take.
    • Network iteration count: how many times is the model on the whole network trained, that is computing an aggregatation of each local model, and used this new model the feed the next local iterations.
